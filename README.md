# Прогнозирование покупок в интернет-магазине

## 📌 Описание проекта

Интернет-магазин проводит рассылки и хочет оптимизировать маркетинговые коммуникации.

**Задача:** Предсказать вероятность покупки клиентом в течение 90 дней на основе истории покупок и поведения в рассылках.

**Метрика качества:** ROC-AUC

**Результат:** Модель достигла ROC-AUC = 0.821 на тестовой выборке

---


## 📊 Данные

| Файл | Описание |
|------|----------|
| `apparel-purchases`  | Покупки: client_id, quantity, price, category_ids, date, message_id |
| `apparel-messages`  | Рассылки: client_id, event, date, created_at |
| `apparel-target_binary`  | Цель: client_id, target (1 — покупка в течение 90 дней) |

---

## 🧠 Созданные признаки

| Признак | Описание |
|---------|----------|
| `recency_days` | Дней с последней покупки |
| `frequency_orders` | Количество заказов |
| `monetary_total` | Суммарная выручка |
| `top_category_share` | Доля покупок в любимой категории |
| `ctr` | Click-Through Rate (клики / отправки) |
| `avg_response_hours` | Среднее время реакции на рассылку (часы) |

---

## 🤖 Модель

- **Алгоритм:** LightGBM (градиентный бустинг)
- **Задача:** Бинарная классификация
- **Метрика:** ROC-AUC

## 💡 Бизнес-рекомендации

1. **Таргетировать рассылки** на клиентов с высоким CTR и быстрой реакцией
2. **Не опираться сильно** на частоту покупок — этот признак оказался наименее важным
